Основные изменения в коде
В проект были добавлены три этапа: генерация входных матриц, их параллельное перемножение и отдельная обработка результата.
Генерация выполняется отдельным Python-скриптом и формирует квадратные матрицы заданного размера в текстовые файлы.
MPI-часть читает эти матрицы, распределяет строки первой матрицы между процессами, передаёт вторую матрицу всем участникам,
а затем собирает частичные результаты на процессе 0.

Для распараллеливания использован метод горизонтального разбиения по строкам. Процесс с рангом 0 загружает данные,
формирует буферы и распределяет части матрицы A через коллективную операцию рассылки по блокам, а матрица B передаётся целиком всем процессам.
После вычисления каждая часть результата собирается обратно на главном процессе. Такой подход удобен для плотного матричного умножения,
так как каждая строка результата зависит от всей матрицы B.

Результат вычислений сохраняется в отдельный файл вида result_NxN_pX.txt.
Дополнительно выполняется обработка результата, которая формирует отдельный файл со сводной информацией по матрице.

Примечание
Инфраструктура: использовалась Windows-среда с Microsoft MPI и компилятор g++.
Запуск выполнялся через mpiexec, а корректность результатов для всех конфигураций подтверждалась статусом SUCCESS.
Для замеров использовались размеры матриц 200, 400, 800, 1200, 1600 и 2000, а также числа процессов 1, 2, 4 и 8.

Выводы
Параллельная реализация матричного умножения на MPI показала устойчивое ускорение при росте числа процессов.
Для всех размеров матриц подтверждена корректность вычислений, что видно по статусу SUCCESS.
На больших размерах матриц выигрышь от распараллеливания выражен сильнее, а относительные накладные расходы MPI становятся менее заметны.

Наиболее удачным оказался вариант с 8 процессами для больших матриц, где время выполнения заметно сократилось по сравнению с однопроцессорным запуском.
Это подтверждает, что выбранная схема разбиения по строкам подходит для плотного матричного умножения и хорошо масштабируется на умеренном числе процессов.
